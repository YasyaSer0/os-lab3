## Завдання для попередньої підготовки

### Glossary of Basic English Terms

**Command** - An instruction entered in the terminal to perform a specific action in the operating system.

**Shell** - A program that interprets user commands and passes them to the operating system for execution.

**Command Line Interface (CLI)** - A text-based interface that allows users to interact with the operating system by typing commands.

**Prompt** - A text displayed in the terminal indicating that the system is ready to receive a command.

**Argument** - Additional information provided to a command (for example, a file or directory name).

**Option (Parameter)** - A modifier that changes the behavior of a command. It usually begins with '-' or '--'.

**Flag** - A short option that enables or disables a specific function of a command.

**Directory** - A folder used to organize and store files and other directories.

**File** - A data storage object in the file system.

**Path** - The location (address) of a file or directory in the file system.

**Environment Variable** - A variable that stores system or user configuration values.

**Alias** - A custom shortcut name for a command.

**History** - A list of previously executed commands.

**Man Page (Manual Page)** - A built-in documentation page for a command, accessed using the 'man' command.

**Export** - A command used to make a variable available to child processes.

**Unset** - A command used to remove a variable.

**Echo** - A command used to display text or variable values in the terminal.

**Execution** - The process of running a command or program.

### Визначення основних понять

Командний інтерпретатор - це програма, яка приймає команди користувача, аналізує їх та передає операційній системі для виконання.

Оболонка - це програмне середовище, яке забезпечує взаємодію користувача з операційною системою через введення команд або графічний інтерфейс.

Команда - це інструкція, яку користувач вводить у командному рядку для виконання певної дії в операційній системі.

### Відповіді на теоретичні питання

**Яку базову інформацію надає рядок запрошення prompt?**  
Рядок запрошення (prompt) показує, що система готова приймати команду. Зазвичай він містить ім’я користувача, назву комп’ютера та поточний каталог. Також за символом ($ або #) можна зрозуміти, звичайний це користувач чи адміністратор.

**Для чого команді потрібні параметри та аргументи?**  
Параметри (опції) змінюють поведінку команди, додають їй додаткові можливості. Аргументи передають команді конкретні дані, наприклад ім’я файлу або папки. Тобто параметри визначають "як виконувати", а аргументи - "над чим виконувати".

**Яке призначення команди ls, які параметри та аргументи вона може мати? Наведіть 3 приклади.**  
Команда `ls` використовується для перегляду вмісту каталогу. Вона може мати різні параметри, наприклад:
- `-l` - показує детальну інформацію про файли,
- `-a` - відображає приховані файли,
- `-h` - показує розмір файлів у зручному форматі.

Приклади:
- `ls` - показати список файлів у поточній папці.
- `ls -l` - показати детальну інформацію.
- `ls -la /home` - показати всі файли (включаючи приховані) у каталозі /home.

**Яким чином можна використати історію команд, які переваги це надає?**  
Історію команд можна переглянути за допомогою команди `history` або клавішами стрілок вгору/вниз. Це дозволяє швидко повторити попередні команди без повторного введення. Це економить час і зменшує кількість помилок.

**Яке призначення команди echo?**  
Команда `echo` використовується для виведення тексту або значення змінної в терміналі. Наприклад, її можна використати для перевірки значення змінної.

**Охарактеризуйте поняття змінної в оболонці Bash, які типи змінних вона підтримує?**  
Змінна в Bash - це іменована область пам’яті, в якій зберігається певне значення. У Bash є локальні змінні (звичайні змінні оболонки) та змінні середовища. Змінні середовища доступні не тільки в поточній оболонці, а й у дочірніх процесах.

**Яке призначення команд env, export та unset?**  
`env` - показує список змінних середовища.  
`export` - робить змінну доступною для інших процесів (перетворює її на змінну середовища).  
`unset` - видаляє змінну.

**Які команди для отримання довідки по командам в терміналі ви знаєте?**  
Для отримання довідки можна використовувати:
- `man` - відкриває повну сторінку довідки,
- `--help` - коротка довідка по команді,
- `info` - розширена документація,
- `help` - довідка для вбудованих команд Bash.

### Відповіді на контрольні запитання:

## 1. Які типи команд існують в оболонці Bash?
   
   У Bash існують:
- вбудовані команди (cd, echo, export);
- зовнішні команди (ls, uname, cal);
- функції, які створює користувач;
- псевдоніми (aliases).
  
## 2. Що таке змінні оточення? Які вони бувають? Як їх можна переглянути?

Змінні оточення - це змінні, які зберігають системну інформацію (наприклад, PATH, HOME, USER).

Бувають:
- локальні (працюють у поточному сеансі);
- глобальні (доступні для всіх процесів).

Переглянути можна командами:
```bash
env
printenv
set
```

## 3. Опишіть змінну $PS1. Як у терміналі переглянути її вміст?

PS1 - це змінна, яка відповідає за вигляд рядка запрошення в Bash.

Переглянути можна так:
```bash
echo $PS1
```

## 4. Як можна змінити значення змінної $PS1? Що при цьому відбудеться? Як змінити її за замовчуванням?

Змінити можна так:
```bash
PS1="MyPrompt> "
```

Після цього зміниться вигляд рядка запрошення.

Щоб зробити це постійно (за замовчуванням), потрібно записати нове значення в файл:
```bash
~/.bashrc
```

## 5. Для чого використовують лапки в Bash?

Лапки використовуються для правильного оброблення тексту та змінних.

- " " - дозволяють підставляти значення змінних;
- ' ' - сприймають текст буквально;
- \ - екранує символ.
  
## 6. Для чого використовують інструкції керування? Які їх види?

Інструкції керування використовують для організації порядку виконання команд.

Види:
- && - виконати, якщо попередня команда успішна;
- || - виконати, якщо попередня неуспішна;
- if - умовний оператор;
- цикли (for, while).

## 7. В чому різниця між символами $ і # у рядку запрошення?

- $ - означає звичайного користувача.
- **#** - означає користувача root (адміністратор).

## 8. Яке призначення команд whereis та locate? Яка між ними відмінність?

- whereis - показує, де знаходиться виконуваний файл програми.
- locate - шукає файли по всій системі за назвою.

Відмінність:
whereis шукає конкретні системні файли,
а locate працює через базу даних і може знайти майже будь-який файл.
